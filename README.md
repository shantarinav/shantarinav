## Продуктовые линейки

### Рапорт
**Рапорт** — local-first продуктовая линейка управленческих дашбордов для анализа CSV/XLS/XLSX-отчетов промышленного предприятия.

**Слоган:** Excel докладывает главное.

Рапорт работает как статический frontend без обязательного backend: файл загружается в браузере, тип отчета определяется автоматически, данные анализируются локально. Дополнительные backend-сервисы используются только как optional extensions.

| Направление | Назначение | Статус |
|---|---|---|
| Рапорт: качество ССЗ | Контроль качества оформления сменно-суточных заданий и доли работ по технологии | Local v1 |
| Рапорт: исполнительская дисциплина | Анализ сроков согласования и исполнения задач в СЭД Tessa | Local v1 |
| Рапорт: контроль печати | Анализ объемов печати, нецелевой печати, нагрузки и точек оптимизации затрат | Local v1 |
| Рапорт: SLA техподдержки | Анализ закрытых заявок, исполнения SLA и качества работы техподдержки | Local v1 |
| Local A3 | Локальный журнал A3-разборов отклонений с ответственными, сроками и статусами | Local v1 |

Ключевые возможности:

- единая страница загрузки отчетов;
- автоматическое определение типа отчета по структуре файла;
- режимы «Руководитель» и «Аналитик»;
- локальная история месячных KPI-снимков;
- локальные A3-разборы отклонений в IndexedDB;
- экспорт/импорт A3-журнала в JSON;
- темная и светлая тема;
- единый визуальный контракт и переиспользуемые shared UI-компоненты;
- optional ИИ-проверка личной печати для дашборда Print через локальный backend/Ollama.

Текущий технологический стек:

- React;
- TypeScript;
- Vite;
- Tailwind CSS;
- shadcn/ui primitives;
- lucide-react;
- IndexedDB / Dexie для локальных A3 и KPI;
- Zod для схем Local A3;
- xlsx для чтения Excel-файлов.

Архитектурный подход:

- static frontend first;
- hash-routing для публикации в любой папке сайта;
- отсутствие обязательной серверной части;
- разделение парсинга, расчетов, хранения и UI;
- route-level lazy loading для дашбордов;
- безопасное локальное хранение: без исходных Excel-файлов и сырых строк отчетов;
- optional backend extensions только для дополнительных локальных усилителей анализа.

Репозиторий: [raport-product-line](https://github.com/shantarinav/raport-product-line)
