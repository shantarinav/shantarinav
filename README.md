## Продуктовые линейки

### Рапорт
**Рапорт** — продуктовая линейка управленческих дашбордов с локальной обработкой CSV/XLS/XLSX-отчетов промышленного предприятия.

**Слоган:** Excel докладывает главное.

Рапорт работает как статическое веб-приложение без обязательной серверной части: файл загружается в браузере, тип отчета определяется автоматически, данные анализируются локально. Дополнительные серверные сервисы используются только как необязательные расширения.

| Направление | Назначение | Статус |
|---|---|---|
| Рапорт: качество ССЗ | Контроль качества оформления сменно-суточных заданий и доли работ по технологии | v1.0 |
| Рапорт: исполнительская дисциплина | Анализ сроков согласования и исполнения задач в СЭД Tessa | v1.0 |
| Рапорт: контроль печати | Анализ объемов печати, нецелевой печати, нагрузки и точек оптимизации затрат | v1.0 |
| Рапорт: SLA техподдержки | Анализ закрытых заявок, исполнения SLA и качества работы техподдержки | v1.0 |
| Локальные A3-разборы | Журнал A3-разборов отклонений с ответственными, сроками и статусами | v1.0 |
| Raport LLM | Необязательный локальный ИИ-сервис для проверки личной печати в Print и черновиков A3-разборов через Ollama | v1.0 |

Ключевые возможности:

- единая страница загрузки отчетов;
- автоматическое определение типа отчета по структуре файла;
- режимы «Руководитель» и «Аналитик»;
- локальная история месячных KPI-снимков;
- локальные A3-разборы отклонений в IndexedDB;
- экспорт/импорт A3-журнала в JSON;
- темная и светлая тема;
- единый визуальный контракт и переиспользуемые интерфейсные компоненты;

Текущий технологический стек:

- React;
- TypeScript;
- Vite;
- Tailwind CSS;
- shadcn/ui primitives;
- lucide-react;
- IndexedDB / Dexie для локальных A3 и KPI;
- Zod для схем A3-разборов;
- xlsx для чтения Excel-файлов.

Опциональная серверная часть:

- Node.js для Raport LLM;
- Ollama для локальных моделей;
- SQLite-кэш только внутри ИИ-сервиса, без обязательной зависимости фронтенда от сервера.

Архитектурный подход:

- сначала статическое веб-приложение;
- hash-routing для публикации в любой папке сайта;
- отсутствие обязательной серверной части;
- разделение парсинга, расчетов, хранения и UI;
- загрузка страниц дашбордов по мере обращения к ним;
- безопасное локальное хранение: без исходных Excel-файлов и сырых строк отчетов;
- необязательные серверные расширения только для дополнительных локальных усилителей анализа; Raport LLM вынесен отдельно от фронтенда и может запускаться на ПК пользователя или в корпоративной сети.
